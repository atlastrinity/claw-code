# iOS Remote Client для Claw-Code - Технічне Завдання (Skill)

## 1. Загальний опис
Мета цього завдання — створити повноцінний управлінський фронтенд для `claw-code` на базі iOS-додатка (Remote Client). Цей додаток буде виступати клієнтом, а існуючий `claw-code` — бекендом.

Додаток повинен надавати:
1. Вікно чату для спілкування з агентом (надсилання промптів, отримання відповідей).
2. Інтерфейс перегляду виконання плану (Roadmap/Task Graph).
3. Сортування та фільтрацію по помилках (failed tasks, exceptions).
4. Перегляд та фільтрацію використаних інструментів (executed tools) під час сесії.
5. Загальний моніторинг робочого процесу (Status & Logs).

## 2. Основні екрани та Дизайн (Premium UI/UX)
Дизайн повинен бути надсучасним, стильним та виглядати як преміальний продукт:
- **Естетика:** Використовуй сучасні патерни (Glassmorphism, плавні градієнти, темна тема - Dark Mode за замовчуванням).
- **Типографіка та Анімації:** Використовуй красиву системну типографіку (San Francisco), мікро-анімації при натисканні на кнопки, плавні переходи (transitions) при появі нових повідомлень чи завдань.
- **Chat View (Чат):** Інтерфейс чату з "бульбашками" повідомлень, підтримкою Markdown, підсвіткою коду та красивим інпутом.
- **Task/Plan Dashboard (Вікно виконання плану):** Ієрархічне відображення `task.md` у вигляді дерева (Graph). Повинні бути чітко позначені виконані `[x]`, активні `[/]` та провалені `[-]` завдання. Можливість розгортати/згортати гілки плану з гарними анімаціями.
- **Terminal/Logs View (Фільтрація логів):** Окремий екран для перегляду технічних логів у стилі терміналу (моноширинний шрифт, кольорове маркування Error/Warning/Info).

## 3. Мережевий рівень (Networking)
Додаток повинен підключатися до локального бекенду `claw-code`.
- **WebSocket:** Використовувати `URLSessionWebSocketTask` для забезпечення двостороннього з'єднання у реальному часі (Real-time).
- **JSON RPC/NDJSON:** Підтримка парсингу потокових даних.
- **Моделі даних (Data Models):**
  - `Message`: текстове повідомлення.
  - `TaskNode`: елемент графа завдань зі статусами (`pending`, `in_progress`, `completed`, `failed`).
  - `ToolCall`: інформація про використаний інструмент.

## 4. Вимоги до коду та Якості (Linters)
- **Фреймворк:** SwiftUI.
- **Архітектура:** MVVM (Model-View-ViewModel).
- **Лінтинг та перевірки:** Весь написаний код повинен перевірятися спеціальними лінтерами (наприклад, SwiftLint) під час збірки проєкту. Ти повинен слідкувати за тим, щоб код відповідав стандартам і не містив ворнінгів.

## 5. Перші кроки для розробки (Task Roadmap)
1. Створити базові моделі для парсингу WebSocket-повідомлень.
2. Реалізувати WebSocket Manager для стабільного підключення та реконнекту.
3. Побудувати преміальні SwiftUI-компоненти для чату.
4. Розробити красивий UI для Task Graph з відображенням ієрархії.
5. Зв'язати фільтри логів зі стрімом даних, додати лінтери у фази збірки (Build Phases).

## 6. CRITICAL AUTONOMY RULE
YOU MUST NOT STOP. Once you start, you must relentlessly execute the entire Task Roadmap sequentially. DO NOT ask the user for permission to proceed to the next step. DO NOT say 'I am ready to continue'. Just execute the next step immediately until everything is completed.

## 7. Інструменти та Тестування (XcodeBuildMCP)
Для розробки під iOS ти маєш доступ до потужного сервера `xcodebuildmcp`, який надає близько 80 різних інструментів для роботи з Xcode та симуляторами.
- Оскільки інструментів дуже багато, використовуй інструмент пошуку, щоб знаходити потрібні команди.
- **ОБОВ'ЯЗКОВО ТЕСТУЙ НА СИМУЛЯТОРІ:** Ти повинен використовувати iOS Симулятор для тестування **кожної написаної функції, кожної кнопки в UI та кожної сторінки дизайну**. 
- Запускай додаток у симуляторі, взаємодій з ним, перевіряй ієрархію UI, і тільки коли переконаєшся, що кнопка працює і дизайн виглядає круто, переходь до наступного завдання.
