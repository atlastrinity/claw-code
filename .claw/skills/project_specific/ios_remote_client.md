# 🛡️ Комплексний тест-драйв: Тестувальний скіл claw-code

## Проєкт: Створення сучасного додатка тривог «SirenUA» (iOS / SwiftUI)

### 🎯 Головна мета

Активувати тестувальний скіл **claw-code** для проектування, кодування, тестування та UI-валідації додатка, який у реальному часі стягує дані про повітряні тривоги в Україні, відображає їх на інтерактивній карті та симулює критичні сповіщення.

---

## 🛠️ Етап 1: Архітектура та Мережевий шар (API)

На цьому етапі **claw-code** має продемонструвати вміння створювати проєкт та працювати з реальними (або імітованими) даними через API (наприклад, модель даних для сервісу _alerts.in.ua_ або відкритих JSON-джерел державних сповіщень).

> **Промпт для claw-code:**
> \*"Задій свій тестувальний скіл. Створи з нуля новий iOS проєкт під назвою 'SirenUA' на SwiftUI. Створи мережевий шар (NetworkManager) на базі Async/Await, який робить запит до API тривог (можеш використати тестовий JSON-ендпоінт або замокати структуру реального API alerts.in.ua). Додаток повинен парсити статус для кожної області України: `active: true/false`, `type: "air_raid"`, `changed: "Date"`. Напиши Unit-тести, які перевіряють:
>
> 1. Правильність парсингу JSON-відповіді.
> 2. Обробку помилок мережі (якщо сервер повернув код 500 або 404).
>    Запусти збірку та тести через `xcode-bridge`. Покажи мені репорт."\*

---

## 🗺️ Етап 2: Інтеграція сучасного UI (Карта та Віджети)

Тут агент має показати знання сучасних фреймворків Apple (зокрема, оновленого **MapKit** для SwiftUI) та логіку зміни станів.

> **Промпт для claw-code:**
> \*"Тепер розроби інтерфейс додатка 'SirenUA'. На головному екрані має бути:
>
> 1. Інтерактивна карта України (MapKit), де області, в яких зараз оголошено тривогу, підсвічуються червоним кольором, а безпечні — зеленим/прозорим.
> 2. Картка швидкого статусу внизу екрана: кількість активних тривог по країні та остання оновлена область.
> 3. Симуляція **Critical Alerts** (критичних попереджень), які пробивають беззвучний режим (запрограмуй локальний тригер для тесту).
>    Переконайся, що проєкт компилюється через `xcode-bridge`. Якщо виникнуть помилки у зв'язці MapKit та SwiftUI — виправ їх самостійно."\*

---

## 📱 Етап 3: Запуск на симуляторі та стрес-тест даних

Перевірка вміння агента керувати пристроєм та імітувати мінливі бойові умови (зміна статусів тривоги).

> **Промпт для claw-code:**
> _"Використовуючи `ios-simulator` MCP, розгорни додаток 'SirenUA' на симуляторі iPhone 16. Запусти додаток. Тепер проведи стрес-тест: підкинь у додаток через мок-дата фейковий сигнал тривоги (наприклад, 'Київська область — ТРИВОГА'). Перевір через логи, чи додаток миттєво відреагував на зміну даних із нашого сервісу тривог."_

---

## 👁️ Етап 4: Ультимативний UI/UX Аудит (Візуальний тест-скіл)

Фінальний етап, де **claw-code** використовує свої мультимодальні можливості для оцінки естетики та юзабіліті додатка в екстремальних умовах (червоний режим небезпеки).

> **Промпт для claw-code:**
> \*"Коли в симуляторі увімкнено режим активної тривоги (карта червона), зроби скріншот екрана через `ios-simulator`. Проаналізуй зображення:
>
> 1. Чи контрастний текст попередження на червоному тлі? Чи легко його прочитати людині у стресовому стані?
> 2. Чи не перекриває картка статусу елементи керування картою (кнопки зуму/геолокації)?
> 3. Чи виглядає інтерфейс сучасним (закруглені кути, правильні Paddings за гайдлайнами Apple Human Interface Guidelines)?
>    Якщо помітиш недоліки, точково виправ SwiftUI-код верстки, перекомпілюй і надай фінальний звіт розробника."\*
