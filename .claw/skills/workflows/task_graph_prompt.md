# Технічне завдання: iOS Remote Management Client для Claw

Ти — AI-агент, що працює в середовищі `claw-code`. Твоя мета: перетворити існуючий iOS-застосунок `ClawController` на повноцінний управлінський фронтенд-клієнт для нашого бекенду.

**ВАЖЛИВО: Обов'язкове використання TaskGraph!**
Перед тим як написати хоч рядок коду, ти **зобов'язаний** створити детальний план дій за допомогою інструменту `TaskGraph`. Розбий це ТЗ на підзадачі ієрархічно. Щойно ти завершуєш крок — оновлюй статус на `completed`. Якщо ідея не спрацювала — став `failed` і створюй нову гілку рішень. 

## 1. Загальний опис
Наразі `ClawController` має заглушки (mock-дані, `Task.sleep` замість реальних запитів) у `RemoteService.swift`. Нам потрібно зробити так, щоб iOS-клієнт підключався до віддаленого сервера по WebSocket, отримував реальні дані про план виконання (task.md), повідомлення чату та логування інструментів. 

*Примітка: Вважай, що WebSocket сервер на стороні Rust уже розробляється. Твоя задача підготувати iOS-додаток.*

## 2. Що потрібно реалізувати в коді (`ClawControllerPackage`)

### 2.1 Мережеві Моделі (`Models/NetworkModels.swift`)
Створи Codable структури для спілкування з бекендом:
- `TaskGraphNode`: `id` (String), `parent_id` (String?), `content` (String), `status` (Enum: pending, in_progress, completed, failed).
- `ChatMessage`: `role` (Enum: user, agent), `content` (String), `timestamp` (Date).
- `ToolLog`: `tool_name` (String), `input` (String), `output` (String), `status` (Enum: success, error).

### 2.2 Оновлення `RemoteService.swift`
- Заміни фейкові `Task.sleep` на реальний `URLSessionWebSocketTask`.
- Додай парсинг вхідних JSON-повідомлень від сервера та оновлення @Observable властивостей.
- Реалізуй методи для надсилання команд через вебсокет.

### 2.3 Інтерфейс чату (`ChatView.swift`)
- Створи UI для чату (ScrollView з бабблами повідомлень, як в iMessage).
- Інтегруй поле вводу для відправки команд через оновлений `RemoteService`.

### 2.4 Менеджер Завдань (`DashboardView.swift`)
- Перетвори цей екран на візуалізатор графа завдань (`TaskGraph`).
- Використай SwiftUI `OutlineGroup` або `List` з відступами для відображення ієрархії `TaskGraphNode`.
- Додай візуальні індикатори статусів (іконки або кольори для Pending, InProgress, Completed, Failed).

### 2.5 Термінал і Логи (`TerminalView.swift`)
- Зроби екран для перегляду логів виконання інструментів.
- Додай UI-елементи (`Picker`, `Toggle`) для фільтрації:
  - "Всі логи"
  - "Тільки помилки"
  - Фільтрація за конкретним інструментом (наприклад, "Тільки Bash" або "Тільки TaskGraph").

## 3. Вимоги до тестування та перевірки
Після внесення змін до коду, ти повинен:
1. Запустити `xcodebuild` або переконатися, що Swift Package успішно компілюється (без синтаксичних помилок).
2. Задокументувати в TaskGraph успішність компіляції.
